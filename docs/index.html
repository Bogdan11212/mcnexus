Minecraft Engineering.

mcnexus is a production-grade library providing asynchronous tools for Minecraft server management. It is designed for performance, reliability, and ease of use.

📦 Installation

Install the stable version from PyPI:

pip install mcnexus

Development Build

Want to try the latest features before they are released? Install directly from the develop branch:

pip install git+https://github.com/Bogdan11212/mcnexus.git@develop
Class

RCONClient

async command(cmd: str) -> RCONResponse

Sends a command. Automatically handles multi-packet responses and ensures synchronization.

async connect()

Manually establishes the connection. Recommended to use async with instead.

async disconnect()

Closes the active connection.

Class

RCONPool

Manage a network of servers concurrently.

add_server(alias, host, port, password, timeout=10.0, auto_reconnect=True)

Registers a server in the pool under a specific alias.

async connect_all() -> Dict[str, bool]

Attempts to connect to every server in the pool simultaneously.

async broadcast(cmd: str) -> Dict[str, Union[RCONResponse, Exception]]

Sends a command to ALL connected servers at once.

get_client(alias: str) -> RCONClient

Retrieves the specific client instance for a given alias.

Class

RCONWatcher

Simulate event streams using periodic command polling.

on_command_output(command, callback)

Executes callback every interval with the result of the command.

on_output_contains(command, substring, callback)

Executes callback only if the result contains the substring.

async start() / async stop()

Controls the execution of the watcher background tasks.

Function

status()

async status(host, port=25565, timeout=5.0, try_legacy=True) -> StatusResponse

Pings a server. Automatically resolves SRV records and falls back to Legacy protocol if needed.

Function

status_bulk()

async status_bulk(servers: List[tuple], timeout=5.0) -> List[StatusResponse]

Queries multiple servers in parallel. servers should be a list of (host, port) tuples.

Class

LogWatcher

Native event engine that parses latest.log in real-time.

on_event(event_type, callback)

Binds an async function to a specific event (e.g., ChatEvent, DeathEvent).

async start(follow_from_end=True)

Starts tailing the log file. If True, it ignores previous logs.

Class

ServerManager

Universal high-level interface to control servers across different control panels.

async set_power_state(panel_type, panel_url, api_credentials, server_id, action)

Sets the power state of a server regardless of the panel type.

Pterodactyl Configuration

api_credentials = {"api_key": "ptlc_..."}

PufferPanel Configuration

api_credentials = {"client_id": "...", "client_secret": "..."}

Unified Example

from mcnexus import ServerManager

# Works for both panels with one method
await ServerManager.set_power_state(
    panel_type="pterodactyl",
    panel_url="https://panel.myhost.com",
    api_credentials={"api_key": "your_key"},
    server_id="abcdef12",
    action="restart"
)
Class

ScheduleManager

Automate server tasks directly on the panel. Tasks run even if your script is offline.

async create_pterodactyl_schedule(...)

Creates a schedule using Cron syntax (minute, hour, etc.).

async add_pterodactyl_task(...)

Adds an action (command, power, backup) to a schedule.

Chain Task Example (Pterodactyl)

from mcnexus import ScheduleManager

# 1. Create a schedule for 3:00 AM daily
sched = await ScheduleManager.create_pterodactyl_schedule(
    panel_url="...", api_key="...", server_id="...",
    name="Nightly Maintenance", hour="3", minute="0"
)

# 2. Add tasks to that schedule
# Task A: Announce restart
await ScheduleManager.add_pterodactyl_task(..., sched["id"], action="command", payload="say Restarting!")
# Task B: Backup (runs 10s later)
await ScheduleManager.add_pterodactyl_task(..., sched["id"], action="backup", payload="*", time_offset=10)
# Task C: Restart (runs 60s after backup)
await ScheduleManager.add_pterodactyl_task(..., sched["id"], action="power", payload="restart", time_offset=60)
Class

StatsManager

Retrieve real-time resource usage statistics from your control panels.

async get_server_stats(panel_type, panel_url, api_credentials, server_id) -> ResourceStats

Returns a unified model containing CPU, RAM, and Network usage.

ResourceStats Model Properties

PropertyDescription
cpu_usageCPU usage percentage (e.g. 50.5).
memory_mbMemory usage in Megabytes.
network_rx_bytesBytes received by the server.
stateCurrent power state (running/off).

Example

from mcnexus import StatsManager

stats = await StatsManager.get_server_stats(
    panel_type="pterodactyl",
    panel_url="...",
    api_credentials={"api_key": "..."},
    server_id="abcdef12"
)
print(f"CPU: {stats.cpu_usage}% | RAM: {stats.memory_mb:.2f} MB")
